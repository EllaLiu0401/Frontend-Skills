## Common Focus Areas

When reviewing code, consider focusing on:

### Code Quality
- Readability and clarity
- Consistent naming conventions
- Proper abstraction levels
- DRY principles

### Correctness
- Logic errors
- Edge cases
- Error handling
- Type safety

### Performance
- Unnecessary re-renders
- Inefficient algorithms
- Memory leaks
- Bundle size impact

### Security
- Input validation
- XSS vulnerabilities
- Authentication/authorization
- Data sanitization

### Accessibility
- Keyboard navigation
- ARIA attributes
- Semantic HTML
- Color contrast

### Maintainability
- Code organization
- Documentation
- Test coverage
- Dependency management

## Tips for Code Review Prompts

1. **Specify scope**: Be clear about what aspects to focus on
2. **Provide context**: Share relevant project requirements or constraints
3. **Include related code**: Sometimes surrounding code context helps
4. **Ask for alternatives**: Request multiple approaches when appropriate
5. **Request explanations**: Ask for reasoning behind suggestions
